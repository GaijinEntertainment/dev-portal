Modes config format

Statistic modes

Modes makes work with user statistic more flexible. Modes is an abstract like a container for statistics, for example:

  • different game types: solo, squad, etc.

  • different countries: USSR, Germany, etc.

  • different locations: Moscow, Berlin, etc.

  • etc (everything you want in your game).

Each mode contains all user statistics and it can be different for each modes. For example: user can have 4 kills in solo mode and 10 kills in squad mode.

You can read about the interaction between the game and the statistics server here: .

Mode format

To use and store statistics, you need to add modes description to modes config, and deploy configs to services.

Mode description format

{
  "name": "solo",             // required

  "leaderboard": "SIMPLE",    // optional, default = "NONE"
  "public": false             // optional, default = false
}
Required fields

  • name (string) – name of the mode, must be unique.

Optional fields

  • leaderboard (string) – type of the leaderboard creating for this mode, possible values:

    • “NONE” – leaderboard will not be generated for this mode.

    • “SIMPLE” – generate leaderboard without groups for this mode.

    • “GROUPING” – generate groups leaderboard for this mode. Group can be used to implement leagues or buckets in leaderboard. See Leaderboard group.

  • public (bool) – determines whether the stats for this mode should be visible to another user.

    Note

    This works only for stats with showForAll = true. To request another user statistics, use action.
